# Wireless Network Troubleshooting Lab (Cisco Packet Tracer)

## Overview
This project demonstrates troubleshooting a wireless connectivity issue using Cisco Packet Tracer in a simulated enterprise network environment.

## Scenario
A client device was unable to connect to a wireless network and had no internet access. The issue required diagnosing both client-side and network-side configurations.

## Troubleshooting Process

1. Verified wireless network (SSID) and connection settings
2. Checked IP configuration using `ipconfig`
3. Tested connectivity using `ping`
4. Identified incorrect or missing IP addressing
5. Reviewed wireless router and DHCP configuration
6. Corrected network settings
7. Re-tested connectivity to confirm resolution

## Tools & Technologies
- Cisco Packet Tracer
- Command Prompt (CLI)
- ipconfig
- ping

## Results
